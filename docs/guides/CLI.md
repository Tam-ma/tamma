# Tamma CLI Guide

Quick-start guide for the `tamma` command-line interface.

## Prerequisites

- **Node.js** 20+
- **pnpm** 9+
- **Git** 2.40+
- **Claude Code CLI** installed and authenticated (`claude --version`)
- A **GitHub** personal access token with `repo` scope
- An **Anthropic API key**

## Getting Started

### 1. Initialize a project

```bash
tamma init
```

The init wizard runs four phases:

1. **Pre-flight checks** — verifies Node, Git, and Claude CLI are available
2. **Configuration wizard** — collects credentials and project settings:
   - GitHub token (masked input, or press Enter to use `GITHUB_TOKEN` env var)
   - Anthropic API key (masked input, or press Enter to use `ANTHROPIC_API_KEY` env var)
   - GitHub owner and repository (auto-detected from git remote)
   - Issue labels, AI model, budget, working directory, approval mode
3. **Post-config checks** — tests GitHub API access, validates Claude CLI, creates `tamma` label, updates `.gitignore`
4. **Done** — prints next steps

The wizard produces two files:

| File | Contents | Permissions |
|------|----------|-------------|
| `tamma.config.json` | Project settings (no secrets) | Default |
| `.env` | `GITHUB_TOKEN` and `ANTHROPIC_API_KEY` | `0600` (owner-only) |

Both `.env` and `.tamma/` are added to `.gitignore` automatically.

If a `.env` file already exists, the wizard merges credentials into it — existing variables are preserved.

### 2. Start the engine

```bash
tamma start
```

The engine loads configuration in layers (later layers override earlier ones):

```
defaults → tamma.config.json → .env / environment variables → CLI flags
```

It then polls for GitHub issues matching your configured labels and processes them through the pipeline:

```
Select issue → Analyze context → Generate plan → Approval → Implement → Create PR → Monitor CI → Merge
```

## Commands

### `tamma init`

Interactive setup wizard. Will not overwrite an existing `tamma.config.json` — delete it first to re-run.

### `tamma start [options]`

Start the engine.

| Flag | Description | Default |
|------|-------------|---------|
| `--config <path>` | Path to config file | `./tamma.config.json` |
| `--dry-run` | Select, analyze, plan, then exit (no implementation) | off |
| `--once` | Process one issue then exit | off |
| `--approval <mode>` | `cli` (manual) or `auto` (approve all plans) | `cli` |
| `-i, --interactive` | Interactively select which issue to process | off |
| `--verbose` | Enable debug logging | off |
| `--debug` | Debug logging + write logs to file | off |

### `tamma status`

Show current engine state.

## Configuration

### `tamma.config.json`

Generated by `tamma init`. Key sections:

```jsonc
{
  "mode": "standalone",
  "logLevel": "info",
  "github": {
    "token": "",           // loaded from .env, not stored here
    "owner": "your-org",
    "repo": "your-repo",
    "issueLabels": ["tamma"],
    "excludeLabels": ["wontfix"],
    "botUsername": "tamma-bot"
  },
  "agent": {
    "model": "claude-sonnet-4-5",
    "maxBudgetUsd": 1.0,
    "allowedTools": ["Read", "Write", "Edit", "Bash", "Glob", "Grep"],
    "permissionMode": "default"
  },
  "engine": {
    "pollIntervalMs": 300000,
    "workingDirectory": ".",
    "maxRetries": 3,
    "approvalMode": "cli",
    "ciPollIntervalMs": 30000,
    "ciMonitorTimeoutMs": 3600000
  }
}
```

### Multi-Agent Configuration (Epic 9)

The `agents` block replaces or augments the legacy `agent` block when you want to use multiple AI providers, role-based assignment, or fallback chains. It is optional; the legacy `agent` block continues to work and is automatically converted at startup.

```jsonc
{
  "agents": {
    "defaults": {
      "providerChain": [
        { "provider": "claude-code", "model": "claude-sonnet-4-5" },
        { "provider": "openrouter", "model": "anthropic/claude-3-5-sonnet", "apiKeyRef": "OPENROUTER_API_KEY" }
      ],
      "allowedTools": ["Read", "Write", "Edit", "Bash", "Glob", "Grep"],
      "maxBudgetUsd": 2.0,
      "permissionMode": "default"
    },
    "roles": {
      "implementer": {
        "providerChain": [
          { "provider": "opencode", "model": "claude-sonnet-4-5" },
          { "provider": "claude-code", "model": "claude-sonnet-4-5" }
        ],
        "maxBudgetUsd": 5.0
      },
      "reviewer": {
        "providerChain": [
          { "provider": "openrouter", "model": "openai/o3-mini", "apiKeyRef": "OPENROUTER_API_KEY" }
        ],
        "maxBudgetUsd": 1.0
      }
    },
    "phaseRoleMap": {
      "CODE_GENERATION": "implementer",
      "CODE_REVIEW": "reviewer"
    }
  }
}
```

See [AGENTS.md](AGENTS.md) for the full schema reference, including all `AgentType` values, `WorkflowPhase` values, provider chain fallback logic, prompt template system, and security features.

### Security Configuration (Epic 9)

The `security` block enables content sanitization, URL validation, and action gating:

```jsonc
{
  "security": {
    "sanitizeContent": true,
    "validateUrls": true,
    "gateActions": true,
    "maxFetchSizeBytes": 10485760,
    "blockedCommandPatterns": [
      "git push --force",
      "npm publish"
    ]
  }
}
```

| Field | Type | Description |
|-------|------|-------------|
| `sanitizeContent` | `boolean` | Strip HTML and detect prompt injection in agent I/O |
| `validateUrls` | `boolean` | Reject private/reserved hosts before outbound requests |
| `gateActions` | `boolean` | Block dangerous shell commands via the action gate |
| `maxFetchSizeBytes` | `number` | Maximum response body size for secure fetch (0–1 GiB) |
| `blockedCommandPatterns` | `string[]` | Additional shell command patterns to block (up to 100, each a valid regex) |

### `.env`

Credentials file written by `tamma init` with restricted permissions (`0600`):

```bash
# Tamma credentials — DO NOT COMMIT
GITHUB_TOKEN=ghp_...
ANTHROPIC_API_KEY=sk-ant-...
```

Loaded automatically at startup via `dotenv`. If you skip a credential during init, it is written as a commented placeholder:

```bash
# GITHUB_TOKEN=ghp_your_token_here
```

Docker secret file support is also available. If `GITHUB_TOKEN_FILE=/run/secrets/github_token` is set, the contents of that file are used instead of the plain `GITHUB_TOKEN` variable.

### Environment Variables

All settings can be overridden via environment variables. Later layers override earlier ones: config file values are overridden by environment variables, which are in turn overridden by CLI flags.

#### Core Configuration

| Variable | Overrides |
|----------|-----------|
| `GITHUB_TOKEN` / `TAMMA_GITHUB_TOKEN` | `github.token` |
| `TAMMA_GITHUB_OWNER` | `github.owner` |
| `TAMMA_GITHUB_REPO` | `github.repo` |
| `TAMMA_BOT_USERNAME` | `github.botUsername` |
| `TAMMA_ISSUE_LABELS` | `github.issueLabels` (comma-separated) |
| `TAMMA_EXCLUDE_LABELS` | `github.excludeLabels` (comma-separated) |
| `TAMMA_MODEL` | `agent.model` |
| `TAMMA_MAX_BUDGET_USD` | `agent.maxBudgetUsd` |
| `TAMMA_PERMISSION_MODE` | `agent.permissionMode` |
| `TAMMA_AGENT_PROVIDER` | `agent.provider` (`anthropic`, `openai`, or `local`) |
| `TAMMA_POLL_INTERVAL_MS` | `engine.pollIntervalMs` |
| `TAMMA_WORKING_DIRECTORY` | `engine.workingDirectory` |
| `TAMMA_APPROVAL_MODE` | `engine.approvalMode` |
| `TAMMA_LOG_LEVEL` | `logLevel` |

#### Security Environment Variables (Epic 9)

These environment variables control the security layer without editing the config file:

| Variable | Overrides | Values |
|----------|-----------|--------|
| `TAMMA_SANITIZE_CONTENT` | `security.sanitizeContent` | `true` or `false` |
| `TAMMA_VALIDATE_URLS` | `security.validateUrls` | `true` or `false` |
| `TAMMA_GATE_ACTIONS` | `security.gateActions` | `true` or `false` |
| `TAMMA_MAX_FETCH_SIZE_BYTES` | `security.maxFetchSizeBytes` | integer bytes |
| `TAMMA_ALLOW_BYPASS_PERMISSIONS` | Enables `permissionMode: bypassPermissions` in task configs | `true` |

#### Provider API Key Environment Variables (Epic 9)

API keys for provider chain entries are resolved at runtime using the `apiKeyRef` field in each chain entry. The variable name is whatever you specify in `apiKeyRef`:

```jsonc
{ "provider": "openrouter", "apiKeyRef": "OPENROUTER_API_KEY" }
```

Tamma reads `process.env["OPENROUTER_API_KEY"]`. If the variable is not set when the factory tries to create the provider, that entry is skipped and the chain moves to the next entry.

Common provider API key variables:

| Provider | Typical Environment Variable |
|----------|------------------------------|
| `claude-code` | Not required (uses Claude subscription auth) |
| `opencode` | Not required (uses local OpenCode process) |
| `openrouter` | `OPENROUTER_API_KEY` |
| `zen-mcp` | `ZEN_MCP_API_KEY` (or whatever you name in `apiKeyRef`) |

## Typical Workflow

```bash
# First time setup
tamma init

# Process all labeled issues continuously
tamma start

# Process one issue then stop
tamma start --once

# Preview what the engine would do without implementing
tamma start --dry-run

# Pick which issue to work on
tamma start --interactive --once

# Auto-approve all plans (no manual review)
tamma start --approval auto
```

## Config-Driven Agent Resolution (Epic 9)

Starting with Epic 9, the CLI uses `normalizeAgentsConfig()` to translate any `TammaConfig` into the canonical `AgentsConfig` format before passing it to the engine. This means the engine always operates against a consistent multi-agent structure regardless of whether the config file uses the new `agents` block or the legacy `agent` block.

The resolution order for the CLI config loader is:

```
defaults → tamma.config.json (agents or agent) → environment variables → CLI flags
```

If both `agents` and `agent` are present in `tamma.config.json`, the `agents` block takes precedence. The legacy `agent` block is used only when `agents` is absent.

The `normalizeAgentsConfig()` function is exported from `packages/cli/src/config.ts` for use by other parts of the system:

```typescript
import { normalizeAgentsConfig } from '@tamma/cli/config';
// or directly from @tamma/shared:
import { normalizeAgentsConfig } from '@tamma/shared';
```

## .env.example

Run `tamma init` or generate a template manually. The full list of supported environment variables is:

```bash
# GitHub — required
GITHUB_TOKEN=
TAMMA_GITHUB_OWNER=
TAMMA_GITHUB_REPO=

# GitHub — optional
TAMMA_BOT_USERNAME=tamma-bot
TAMMA_ISSUE_LABELS=tamma
TAMMA_EXCLUDE_LABELS=wontfix

# Agent (legacy single-provider config)
TAMMA_MODEL=claude-sonnet-4-5
TAMMA_MAX_BUDGET_USD=1.00
TAMMA_PERMISSION_MODE=default
# TAMMA_AGENT_PROVIDER=anthropic

# Security (Epic 9)
# TAMMA_SANITIZE_CONTENT=true
# TAMMA_VALIDATE_URLS=true
# TAMMA_GATE_ACTIONS=false
# TAMMA_MAX_FETCH_SIZE_BYTES=10485760
# TAMMA_ALLOW_BYPASS_PERMISSIONS=false

# Engine
TAMMA_POLL_INTERVAL_MS=300000
TAMMA_WORKING_DIRECTORY=.
TAMMA_APPROVAL_MODE=cli

# Logging
TAMMA_LOG_LEVEL=info

# Claude CLI agent — required for claude-code provider
ANTHROPIC_API_KEY=

# Provider chain API keys (Epic 9) — add as needed
# OPENROUTER_API_KEY=
# ZEN_MCP_API_KEY=
```
